# Interpretable Factor Decomposition for Decision Intelligence in China's A-Share Market

Reproducibility code for the paper **"Interpretable Factor Decomposition for Decision Intelligence in Large-Scale Financial Markets: Evidence from China's A-Share Market."**

The work builds an **interpretable machine-learning pipeline** that decomposes
cross-sectional equity-return predictability into auditable factor
contributions. An XGBoost model predicts next-month outperformance
(beat-the-cross-sectional-median), and **TreeSHAP** attributes that prediction
across 13 firm characteristics. An **ablation study** then cross-validates the
SHAP ranking and surfaces feature *substitutability* structure that neither
method reveals in isolation.

| Headline result (55 out-of-sample months) | Value |
|---|---|
| Mean AUC | **0.547** [95% CI 0.532, 0.563] |
| Long–short spread | **+2.38 %/mo** [1.44, 3.34] |
| Annualized Sharpe | **2.23** [1.38, 3.36] |
| Newey–West *t* (L-S mean ≠ 0) | 5.94, *p* < 0.001 |
| Carhart-4 alpha | **+2.31 %/mo** (*t* = 7.48) |
| SHAP share — behavioral vs. valuation | **58.2 % vs. 10.7 %** |

## ⚠️ Read this first

- **No raw data is distributed.** The market data is pulled from the free,
  key-less public APIs **[baostock](http://baostock.com)** and
  **[akshare](https://akshare.akfamily.xyz)** and is large (the daily panel
  alone is ~700 MB). Re-run scripts `03`–`06` to rebuild it locally. See
  [Data](#data-sources) and [Reproduction](#reproducing-the-results).
- **Only code is distributed** — no data, result tables, or figures. All of
  those are generated by running the scripts into `output/` and `figures/`
  (both git-ignored). The paper's headline numbers are summarized in the
  table above.
- Paths inside the scripts resolve **relative to the repo root**, so the code
  runs anywhere (the original absolute paths have been normalized).
- `09_paper1_review_fixes.py` is the **final analysis** that matches the paper.
  Scripts `07` and `08` are earlier iterations and are kept for provenance; their
  outputs (`*_v1`/`*_v2`) are superseded by the `*_v3` results.

## Data sources

| Source | Used for | Auth |
|---|---|---|
| **baostock** | daily prices, turnover, valuation (PE/PB/PS/PCF TTM), quarterly profit (ROE, NP margin), industry codes | free, no key |
| **akshare** | CSI All-Share (000985) constituent list with inclusion dates | free, no key |

- **Universe:** all CSI All-Share constituents, **2009-01-01 → 2019-07-30**,
  ST-flagged securities excluded → **3,632 → 3,199 stocks**, 116 months,
  **254,854 stock-month observations**.
- **Monthly panel** is built by aggregating daily data per stock-month; months
  with fewer than 10 trading days are dropped and features are winsorized at the
  1st/99th percentile within each cross-section.

> baostock/akshare are community-maintained and their schemas can drift. If a
> column is missing or renamed, adapt the `FIELDS` / column lists at the top of
> the relevant pull script.

## Setup

Python 3.8+.

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

`baostock` and `akshare` are only required for the data-pull scripts (`03`–`06`
and `probe_*`). If you already have the data, the analysis scripts need only the
core scientific stack + `xgboost` + `shap` + `statsmodels`.

## Reproducing the results

Run from the **repo root**. All scripts read/write `./output/` and `./figures/`.

```bash
# 1. Pull raw data (hours; ~3,600 stocks; resumable via per-stock CSV cache)
python scripts/03_paper1_constituents.py     # -> output/paper1_stock_universe.csv
python scripts/04_paper1_daily_factors.py    # -> output/paper1_daily_factors.csv (~700 MB)
python scripts/05_paper1_quarterly_fundamentals.py
python scripts/06_paper1_industry_codes.py

# 2. Final analysis (builds panel, trains XGBoost, SHAP, ablation, factor alpha,
#    per-industry SHAP, transaction-cost sensitivity, bootstrap CIs, Newey-West)
python scripts/09_paper1_review_fixes.py      # -> output/paper1_v3_*.csv, figures/fig_v3_*.png

# 3. Baselines
python scripts/run_lr_baseline.py             # logistic-regression walk-forward
python scripts/run_additional_baselines.py    # float-cap & industry-neutral portfolios
```

> **Known dependency gap.** `run_additional_baselines.py` reads
> `output/paper1_monthly_panel_cache.csv` and expects a `mean_mcap` (market-cap)
> column. That cache is an intermediate **not** written by the numbered scripts.
> To use this baseline, save script `09`'s monthly panel — which computes
> `market_cap = last_close × totalShare × 10,000` — as that cache (renaming the
> column to `mean_mcap`). Otherwise skip it; it only feeds the construction-scheme
> comparison (paper Table III-B), not the main results.

## Limitations

*Stated in the paper (Section V):*

- Factor alphas are computed against **self-constructed** Chinese factors, not
  published factor returns (e.g. Liu–Stambaugh–Yuan).
- Quarterly fundamentals are merged **without a publication lag**, introducing a
  bounded look-ahead bias (fundamentals account for 14.7 % of SHAP attribution,
  which bounds the exposure).
- The stock universe excludes delisted names → **survivorship bias**.
- SHAP is computed **in-sample**; no multiple-testing correction is applied
  across the 55-industry analysis.

*Code-level caveats:*

- `07`/`08` are superseded iterations; `09` is final.
- `run_additional_baselines.py` depends on an intermediate panel cache not
  produced by the numbered scripts (see note under *Reproducing the results*).
- Results depend on third-party API data that may change over time.
